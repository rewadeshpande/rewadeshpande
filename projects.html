  1. Malicious node detection in Wireless sensor Networks

    2. Summary:

    This project introduces a pioneering approach to enhance the security of wireless sensor networks (WSNs) by leveraging blockchain and smart contract technologies. Traditional methods for detecting malicious nodes often overlook data safety, prompting the need for a more robust solution. The proposed model integrates blockchain's decentralized and secure ledger with smart contracts' automation capabilities, ensuring data integrity, traceability, and efficient detection of malicious nodes. Factors such as Delayed Transmission (DT), Forwarding Rate (FR), and Response Time (RT) are evaluated, and the resulting Node Communication Quality (NCQ) is compared against a threshold to identify malicious nodes. The implementation, carried out using MATLAB/OPNET for WSN simulation and Ethereum for smart contracts, demonstrates a secure, efficient, and reliable solution for safeguarding wireless sensor networks.

    Future Directions:

    Future work could focus on scalability and real-world deployment, exploring further optimizations and broader applications of the proposed blockchain-based model in the evolving landscape of the Internet of Things (IoT) and sensor networks. The project's key contributions lie in ensuring data safety, traceability, and improved reliability in the detection process, laying the groundwork for more secure and efficient wireless sensor networks in the future.

  2. Chicago Transit Authority (CTA) Data Analysis

    3. Summary:

    This project focuses on optimizing Chicago's transit routes by integrating big data tools and analytics. The primary goal is to leverage diverse datasets, including ridership data, crime statistics, and weather conditions, for enhancing the efficiency and safety of the Chicago Transit Authority (CTA) services. The project aims to establish an Extract, Transform, Load (ETL) pipeline utilizing AWS Lambda, Amazon S3, and Amazon Redshift, followed by visualization in Tableau. Key data processing steps involve removing empty values and duplicates, ensuring uniformity in date and time formats, and combining multiple CSV files into a cohesive dataset. The datasets encompass CTA ridership data, crime data, and additional sources such as holiday information and weather data. The ultimate objective is to derive meaningful insights into optimal transit routes based on ridership patterns, weather conditions, and crime occurrences.

    Technologies Utilized:

    • AWS Lambda: Automated the extraction of data from Amazon S3 and facilitated seamless data transfer.
    • Amazon S3: Acted as a centralized storage repository for raw data, enabling efficient data extraction.
    • Amazon Redshift: Served as the data warehouse, offering a scalable solution for handling complex transportation datasets.
    • Tableau: Provided a robust platform for data visualization, offering insights through interactive and visually appealing dashboards.

    Implementation Phases:

    • Data Export Automation with Lambda: Automated the export of CTA data to Amazon S3, ensuring a continuous flow of updated information.
    • Loading Data into Redshift: Efficiently loaded data from Amazon S3 into Redshift, dynamically creating tables for organized storage.
    • VPC Connection: Established a secure connection between Lambda and Redshift through Amazon VPC, enhancing data pipeline security.
    • Data Visualization with Tableau: Integrated Redshift data seamlessly with Tableau, providing stakeholders with visually appealing and informative dashboards.

    In conclusion, this integrated data analytics project represents a strategic endeavor to optimize Chicago's transit routes holistically. By harnessing the power of big data tools and analytics, we have successfully processed diverse datasets encompassing ridership, crime, and weather information. The establishment of a robust ETL pipeline using AWS Lambda, Amazon S3, and Amazon Redshift, followed by visualization in Tableau, underscores our commitment to deriving meaningful insights. The meticulous data processing steps ensure the integrity and uniformity of the datasets. This project not only lays the foundation for data-driven decision-making in the enhancement of Chicago Transit Authority (CTA) services but also highlights the potential for continued exploration and improvement in the efficiency and safety of public transportation systems. The integration of various data sources and technologies signifies a comprehensive approach to address the complexities inherent in urban transit planning, paving the way for informed and optimized transit routes.

  4. Comprehensive Database Management System Implementation

    5. This project presents a comprehensive implementation of a Database Management System (DBMS) by addressing three fundamental components: Storage Manager, Buffer Manager, and Record Manager.

    In the first phase, a robust storage manager was developed to handle file creation, opening, and closing operations. Methods for reading and writing fixed-size pages from/to a file on disk were implemented. Crucial file information such as the number of pages, current page position, and file descriptor were maintained. The introduction of error codes allowed for effective handling of scenarios like non-existing pages and file not found.

    The second phase involved the design and implementation of a versatile buffer manager responsible for managing pages in memory through buffer pools. Support for multiple buffer pools, each associated with a specific page file, was implemented, along with page replacement strategies (e.g., FIFO and LRU). Proper handling of dirty pages, along with maintaining statistics about IO operations, ensured efficient memory management. Additional functionalities for pinning and unpinning pages, marking pages as dirty, and forcing pages back to disk were developed.

    The final phase focused on creating a comprehensive record manager capable of handling tables with fixed schemas. Functionalities for inserting, deleting, updating, and scanning records in a table were implemented. An efficient record representation, page layout, and record IDs were defined to optimize data management. Free space management, schema definition, and consideration of data types in records were addressed.
